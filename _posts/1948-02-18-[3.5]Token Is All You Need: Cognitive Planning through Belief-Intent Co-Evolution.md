# [3.5] Token Is All You Need: Cognitive Planning through Belief-Intent Co-Evolution

- Authors: Shiyao Sang
- [arXiv Link](https://arxiv.org/abs/2511.05540v2)
- [PDF Link](https://arxiv.org/pdf/2511.05540v2.pdf)

## Subfields
 端到端自动驾驶 / 轨迹规划 / 世界模型
## Reason for Interest

【主要问题：实验基准严重错误】
1. 比较逻辑谬误：论文将自己在 nuPlan 数据集上的测试结果（ADE/FDE）直接与 UniAD、VAD 等方法在 nuScenes 数据集上的结果进行数值对比。由于两个数据集的场景分布、坐标尺度和难度完全不同，这种跨数据集的数值比较在科学上是无效的，无法证明其优越性。
2. 数据规模极小：作为一个宣称“世界模型”的方法，仅在 nuPlan 的 720 个场景（约 1.1 万样本）上进行训练和验证，数据量远低于行业标准（通常需数百万帧），不足以证明模型的泛化能力或“认知”涌现。
3. 任务定义不公：该方法使用光栅化的 BEV 语义地图（假设感知完美）作为输入，属于单纯的规划任务；却与处理原始传感器数据（存在感知噪声）的端到端方法（如 UniAD）进行对比，且未在同等输入条件下比较基准模型。
4. 创新点：提出“规划无需像素级重建”的观点具有一定启发性，且稀疏 Token 的设计符合高效推理的趋势，但由于实验设计的根本性缺陷，无法支撑其 SOTA 结论。
